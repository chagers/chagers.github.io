Forking/Cloning

    Forking:

  1. Navigate to DBC's repository that you need, click on the Fork button on the top right.

  2. Remember to Enable Issues in your repository's settings so you can give/receive feedback from other students!

    Cloning:

  1. Go to your command line, find a place where you want to clone this repository. DO NOT clone it into another git repository. Scary things will happen!

  2. Once in the directory where you want it cloned, type in: $ git clone <COPIED_URL>

  3. Hit enter and let it do its work. Now you've cloned the fork!

Setting Remotes

After setting up your local repository, you'll need to configure Git to sync your fork with the cloned repository (remember, you can't push changes to the original DBC repo!).

  1. Via your command line, navigate into the location of your cloned fork.

  2. Type $ git remote -v to see your current configured remotes

  3. To add a new remote, type $ git remote add <name> <url>

  4. Make sure your 'upstream' remote is set to DBC's repo so you can pull/fetch and merge new curriculum changes

  5. Make sure your 'origin' remote is set to your GitHub forked repository

  6. To verify that your new repositories are set up, type $ git remote -v again to check your remotes

Pull Requests

You'll want to use Pull Requests when collaborating on work and sending changes to a GitHub repository.

  1. Make a New Branch for the changes you're making (i.e. not master): $ git co -b new-branch-name (replace "new-branch-name" with the name of the branch you want)

  2. Push your new branch to GitHub
    $ git push origin new-branch-name

  3. On GitHub, go to your fork, and click on 'Pull Request' in the gray nav bar

  4. Hack the pull request URL: https://github.com<YOUR_USERNAME>/phase-0-unit-1/compare/master...

  5. Once you're on the correct branch ('base: master') you may want to put a comment in on what you changed, if not, just press "Create Pull Request"

  6. Merge it, confirm it, delete the branch

Undoing a Merge

Merge Conflicts:

A merge conflict happens when two branches change the same file and try to merge them together. To fix them do the following:

  1. Type $ git status to see what is going on, then open the doc and see what a conflict looks like

  2. You'll see some notation added to the doc that will look something like this:

  3. You will have to choose what you want to keep, some_code or other_code. Delete everything else ( ====, <<< and >>> ) that you do not need. Add and commit, merge conflict resolved!

Undoing Merges by Resetting Commits:

  1. Type in $ git log to check which commit is one prior to the merge conflict on the current branch. Then you can reset to that commit using:
    $ git reset [commit_number]

  2. To reset ALL commits on the branch, use:
    $ git reset --merge ORIG_HEAD

  3. To go back 5 commits, use:
    $ git reset --hard HEAD~5
